# 전체 문서정리

- 전체 문서정리의 컨텐츠는 문서정리, 개발문서, 기술문서 그리고 용어문서 4가지 서브폴더 구조로 구성되어 있다.
- 해당 페이지는 문서정리 페이지로 전체 폴더에 대한 설명과 전체 서브 폴더 구조에 대해 확인할 수 있다.
- 각 서브폴더는 상단의 탭(PC), 혹은 우측상단의 햄버거바(Mobile)로 넘어갈 수 있다.
- 홈 화면으로 돌아가려면 좌측위에 `개발새발`을 클릭하면 기존의 홈 화면으로 넘어갈 수 있다.

## 폴더구조 요약 및 정리

- 이 문서 페이지를 제외한 나머지 페이지는 아래와 같이 구성되어 있다.
  - 개발문서
    - 개발 문서정리의 컨텐츠는 지금까지 했던 프로젝트들이 기술되어 있다.
      - 일반적으로 소개와 설명이 존재한다.
    - ThridParty를 포함한 더 명시해야 될 내용이 있다면 정리했다.
    - 단독 프로젝트들은 회사에서 진행한 프로젝트이고 대학교때 진행하거나 모각코 혹은 단독으로 진행한 프로젝트는 ToyProjects에 있다.
    - 개발에 대한 구체적인 내용은 [홈페이지의 프로젝트 탭](/project)으로 가서 개발문서 바로가기를 선택한다.
      - GitHub Repository의 권한 Public으로 설정되어있는 경우 GitHub로 연결된다.
      - GitHub Repository의 권한 Private로 설정되어있는 경우 개발문서의 해당 프로젝트로 연결된다.
      - 이때 GitHub로 연결되지 않는 내용이 존재하고 이는 아래와 같은 이유가 있다.
        - 해당 프로젝트가 회사에서 개발한 내용이다.
        - 해당 프로젝트가 회사에서 개발하진 않았지만 공개하지 않기로 한 내용이다.
  - 기술문서
    - 기술문서 문서정리의 컨텐츠는 언어를 제외한 프레임워크, 기술등 개발과 관련된 문서들이 서브폴더 구조로 구성되어 있다.
    - 서브폴더는 JS기반 프레임워크 폴더, 모바일 프레임워크, 서버, 데이터베이스로 구성되어있다.
    - 서브폴더 하위 폴더는 기본적으로 개발환경 설정, 기술문서 정리, 발생이슈로 구성되어있다.
    - 배포가 가능한 경우에 프로젝트를 배포하는 간단한 방법이 추가되어있다.
    - 서로 연결이 필요한 경우를 연결에 필요한 각각의 설정이 따로 정리되어 있다.
    - 추가적으로 개인적으로 학습한 내용 정리되어 있다.
    - 개발을 하며 다시 써야할 것같은 내용들 혹은 처음 본 것들 정리되어 있다.
    - 학습한 내용은 아니지만 기록이 필요하다고 느낀 것들을 정리되어 있다.
  - 용어문서
    - 실제 언어들을 학습하거나 사용하며 알게된 용어들을 정리했다.
    - 개발 플랫폼 혹은 프레임워크를 다루며 알게된 언어 및 용어들을 정리했다.
    - 서버 용어, 보안 용어 등 생소한 언어들을 따로 정리했다.
    - 정보처리기사나 정보보안기사와 같은 공부를 하며 알아야할 용어들을 정리했다.
- 나머지 서브폴더 내부는 각각 서브폴더 명에 해당하는 컨텐츠로 구성되어있다.
- 서브폴더의 세부 정보는 서브폴더 아래 소개에 정리되어있다.
