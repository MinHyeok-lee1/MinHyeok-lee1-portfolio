# I-PIE

- 배수로 탐사로봇으로 [조작 웹사이트](https://hyundai.artwa.kr/)에서 제어
- 로봇에서 촬영된 영상을 웹사이트에서 확인하며 제어가능
- 배수로에서 로봇이 넘어졌을 때 실린더를 사용하여 원래대로 돌아오는 전복복귀 기능이 있음

## 구축 환경

## 구축 환경

- front-end: VSC(VisualStudioCode)와 Svelte를 사용
- back-end: VSC(VisualStudioCode)와 Nest.js를 사용
- database: VSC(VisualStudioCode)와 MongoDB를 사용
- MAIN-MCU: 오렌지 파이와 아두이노로 구성되어 있음

## 구성

I-PIE는 다음과 같이 4종의 모듈로 구성된다.

- front-end: UI 페이지 구성 및 로봇의 제어, 상태 확인 등
- back-end: DB 접근 등의 API 제공을 담당
- database: 로그, 로봇의 상태 저장
- MAIN-MCU: 로봇의 동작, 영상 촬영 등 서버와 상호작용을 함

## 모듈별 메뉴얼

### front-end

사용자와 로봇의 인터렉션을 수행하는 UI, UX

1. 로그인 페이지
2. 로봇 선택 페이지
3. 해당 로봇의 페이지
   - 대시보드 페이지 (모든 로봇 상태 요약)
   - 제어 페이지 (영상, 조작 버튼 출력)
   - 상태 페이지 (로봇의 상태 페이지)
   - 센서 페이지 (로봇의 센서 상태 페이지)

### back-end

DB 접근 등의 API 제공을 담당하며 Front-end, MAIN-MCU, database와 모두 인터렉션하는 서버

### database

로그, 로봇의 상태를 저장하며 back-end에서 주로 데이터를 Read함

### MAIN-MCU

전체 메인 코드의 흐름을 담당

- 로봇으로 들어오는 모든 input, output을 관리
- 전체적인 로봇의 흐름을 제어함
