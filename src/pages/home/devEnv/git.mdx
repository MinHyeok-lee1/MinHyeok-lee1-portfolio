# Github 활용

Git 명령어를 이용하여 VSC에서 Github와 상호작용하는 방법을 기술한 문서이다.

설치는 하단 링크에서 가능한다.

## 자주 쓰이는 Git 명령어 정리

### 1. 사용자 이름 및 email 설정

- **git config --global**: 사용자 이름 및 email 전역 설정

```bash
git config --global user.name "Your Name"
git config --global user.email "your_email@example.com"
```

### 2. 저장소 초기화 및 설정

- **git init**: 새로운 Git 저장소 초기화
  ```bash
  git init
  ```
- **git clone [URL]**: 원격 저장소 복제
  ```bash
  git clone https://github.com/user/repo.git
  ```

### 3. 파일 추적 및 상태 확인

- **git status**: 현재 상태 확인 (변경 사항, 스테이징 상태 등)
  ```bash
  git status
  ```
- **git add [파일명]**: 파일을 스테이징 영역에 추가
  ```bash
  git add filename.txt
  ```
- **git add .**: 현재 디렉토리의 모든 변경 사항을 스테이징
  ```bash
  git add .
  ```

### 4. 커밋 및 로그

- **git commit -m "[메시지]"**: 스테이징된 변경 사항을 커밋
  ```bash
  git commit -m "Initial commit"
  ```
- **git log**: 커밋 히스토리 확인
  ```bash
  git log
  ```

### 5. 원격 저장소와의 상호작용

- **git remote add origin [URL]**: 원격 저장소 추가
  ```bash
  git remote add origin https://github.com/user/repo.git
  ```
- **git push -u origin master**: 변경 사항을 원격 저장소에 푸시
  ```bash
  git push -u origin master
  ```
- **git pull**: 원격 저장소에서 최신 변경 사항을 가져와 병합
  ```bash
  git pull
  ```

### 6. 브랜치 관리

- **git branch**: 현재 브랜치 목록 확인
  ```bash
  git branch
  ```
- **git branch [브랜치명]**: 새로운 브랜치 생성
  ```bash
  git branch new-feature
  ```
- **git checkout [브랜치명]**: 다른 브랜치로 전환
  ```bash
  git checkout new-feature
  ```
- **git checkout -b [브랜치명]**: 브랜치 생성 후 전환
  ```bash
  git checkout -b new-feature
  ```
- **git merge [브랜치명]**: 다른 브랜치를 현재 브랜치에 병합
  ```bash
  git merge new-feature
  ```

### 7. 되돌리기 및 리셋

- **git reset [파일명]**: 파일을 스테이징에서 제거
  ```bash
  git reset filename.txt
  ```
- **git reset --hard [커밋 해시]**: 지정한 커밋 상태로 되돌리기 (변경 사항 삭제)
  ```bash
  git reset --hard a1b2c3d4
  ```
- **git revert [커밋 해시]**: 지정한 커밋을 되돌리는 새로운 커밋 생성
  ```bash
  git revert a1b2c3d4
  ```

### 8. 기타 유용한 명령어

- **git stash**: 작업 중인 변경 사항을 임시로 저장
  ```bash
  git stash
  ```
- **git stash pop**: 임시로 저장한 변경 사항을 복원
  ```bash
  git stash pop
  ```
- **git tag [태그명]**: 커밋에 태그 추가
  ```bash
  git tag v1.0
  ```
- **git fetch**: 원격 저장소의 최신 상태를 로컬로 가져오기 (병합은 하지 않음)
  ```bash
  git fetch
  ```
- **git diff**: 변경 사항 비교
  ```bash
  git diff
  ```

이 기본적인 Git 명령어들을 통해 저장소를 관리하고, 버전 관리를 쉽게 할 수 있다.

## Commit Message Convention

- **🎉 Initial commit:** 프로젝트 초기 설정 및 파일 추가
- **🔖 Version tag:** 특정 버전 또는 릴리스에 태그 추가
- **✨ New feature:** 새로운 기능이나 기능성 추가
- **🐛 Bugfix:** 기존 코드의 버그 수정
- **🗃️ Metadata:** 메타데이터 파일 업데이트 (예: 설정 파일)
- **📚 Documentation:** 프로젝트 문서 추가 또는 업데이트
- **💡 Documenting source code:** 소스 코드 주석 및 문서화 추가/업데이트
- **🏎️ Performance:** 코드 성능 최적화
- **💄 Cosmetic:** UI 스타일, 레이아웃 등 시각적 변경
- **✅ Tests:** 테스트 추가 또는 업데이트
- **🔨 Make a test pass:** 기존 테스트 통과를 위한 코드 수정
- **⚡️ General update:** 코드의 일반적인 업데이트 (소소한 개선, 정리 등)
- **🎨 Improve format/structure:** 코드 형식 또는 구조 개선 (기능 변경 없이)
- **♻️ Refactor code:** 코드 리팩토링 (기능 변경 없이)
- **🔥 Removing code/files:** 불필요한 코드나 파일 삭제
- **💚 Continuous Integration:** CI 관련 수정 및 업데이트
- **🔒 Security:** 보안 강화 및 취약점 개선
- **⬆️ Upgrading dependencies:** 의존성 업그레이드
- **⬇️ Downgrading dependencies:** 의존성 다운그레이드
- **👕 Lint:** 린팅 문제 수정
- **👽 Translation:** 번역 추가 또는 업데이트
- **✏️ Text:** 텍스트 수정 (예: 오타 수정)
- **🚑 Critical hotfix:** 긴급한 핫픽스 적용
- **🚀 Deploying stuff:** 프로젝트 배포
- **🍏 Fixing on MacOS:** macOS 관련 문제 해결
- **🐧 Fixing on Linux:** Linux 관련 문제 해결
- **🏁 Fixing on Windows:** Windows 관련 문제 해결
- **🚧 Work in progress:** 진행 중인 작업 커밋
- **👷 Adding CI build system:** CI 빌드 시스템 추가 또는 업데이트
- **📈 Analytics or tracking code:** 분석 또는 추적 코드 추가/업데이트
- **➖ Removing a dependency:** 불필요한 의존성 제거
- **➕ Adding a dependency:** 새로운 의존성 추가
- **🐋 Docker:** Docker 설정 추가 또는 업데이트
- **🔧 Configuration files:** 설정 파일 추가 또는 업데이트
- **📦 Package.json in JS:** JavaScript 프로젝트의 패키지 관리 파일 수정
- **🔀 Merging branches:** 브랜치 병합
- **💩 Bad code / need improv.:** 개선이 필요한 코드 커밋
- **⏪ Reverting changes:** 이전 커밋 되돌리기
- **💥 Breaking changes:** 호환성을 깨는 변경 사항 도입
- **👌 Code review changes:** 코드 리뷰 피드백 반영
- **♿️ Accessibility:** 접근성 개선
- **🚚 Move/rename repository:** 파일이나 디렉터리 이동/이름 변경

이 컨벤션을 통해 Git 커밋 메시지를 일관성 있게 관리할 수 있다.
