# 구문 강조

###### Nextra는 Shiki를 사용합니다.(새 탭에서 열림)빌드 시 구문 강조를 수행합니다.

###### 매우 안정적이고 성능이 좋습니다. 예를 들어 Markdown 파일에 다음을 추가합니다.

```js
console.log("hello, world");
```

## 인라인 코드

Inlined syntax highlighting is also supported `let x = 1{:jsx}` via:

## 라인 강조

```js {1,4-5}
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}
```

## 하위 문자열 강조 표시

```js /useState/
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}
```

## Next.js 이미지

Next.js 이미지를 사용하는 표준 방법 (새 탭에서 열림)MDX 내부는 구성 요소를 직접 가져오는 것입니다.

<Image src="/example.jpeg" alt="ERROR" width={500} height={500} />

정적 이미지
Nextra는 Markdown 구문을 사용하여 정적 이미지 가져오기를 자동으로 최적화한다..
이미지의 너비와 높이를 지정할 필요는 없으며![]() Markdown 구문을 사용하면 된다.

![Hello](/example.jpeg)

- 이렇게 하면 공용 폴더 안에 demo.png 파일이 로드되고 Next.js \<Image>로 자동 래핑된다.

- 공개를 통해 호스트하지 않으려면 \!\[](.../public/demo.png)를 사용하여 상대 경로에서 이미지를 로드할 수도 있다.

- Next.js Image를 사용하면 레이아웃 이동이 없으며 영상을 로드할 때 기본적으로 흐릿한 아름다운 자리 표시자가 표시된다.
