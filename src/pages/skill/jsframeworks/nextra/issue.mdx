# Nextra 이슈 정리

## Nextra 폴더 구조 변경 이슈(2023-07-04)

<FileTree.Folder name="Database" defaultOpen>
  <FileTree.Folder name="MongoDB" defaultOpen>
    <FileTree.File name="개발환경 설정" />
    <FileTree.Folder name="기술문서 정리" defaultOpen></FileTree.Folder>
    <FileTree.File name="발생이슈 정리" />
    <FileTree.File name="_meta.json" />
  </FileTree.Folder>
  <FileTree.File name="_meta.json" />
  <FileTree.File name="mongodb.mdx" />
</FileTree.Folder>

- 초기구조
  - 위와 같은 폴더 구조에서 아래와 같이 `_metajson.mdx`을 구성했었다.

```json filename="Database/_metajson"
{
  "mongodb": "MongoDB"
}
```

- 그러면 MongoDB 폴더를 클릭하면 mongodb.mdx에 있는 내용이 index의 내용처럼 표시된다.
- 하지만 첫 로드시 시간이 지체되면 이후 내부 파일, 폴더를 클릭할 때까지 로딩이 되지않는 문제를 발견했다.
- 때문에 아래와 같이 폴더구조를 변경하였다.

<FileTree.Folder name="Database" defaultOpen>
  <FileTree.Folder name="mongodb" defaultOpen>
    <FileTree.File name="mongodb.mdx" />
    <FileTree.File name="개발환경 설정" />
    <FileTree.Folder name="기술문서 정리" defaultOpen></FileTree.Folder>
    <FileTree.File name="발생이슈 정리" />
    <FileTree.File name="_meta.json" />
  </FileTree.Folder>
  <FileTree.File name="_meta.json" />
</FileTree.Folder>

- 이후 `Database/mongodb/_metajson`을 `mongodb.mdx`와 소개라는 이름으로 매칭시켜준다.

```json filename="Database/_metajson"
{
  "mongodb": "소개",
  "settings": "개발환경 설정",
  "cookbook": "기술문서 정리",
  "issue": "발생이슈 정리"
}
```

- Database 폴더뿐만이 아니라 기술문서에 있는 모든 문서의 구조를 위 방법과 같이 변경하였다.
